# Security Policy

## Supported Versions

The current public release line is supported for security fixes.

## Reporting a Vulnerability

Please report vulnerabilities privately to `<security-contact@example.com>`.

Do not open public issues containing secrets, raw clinical data, API keys, patient identifiers, or exploit details.

## Data Handling

- Do not commit `.env` files or API keys.
- Do not publish unanonymized prescription images.
- Use opaque black redaction boxes for example images.
- Keep benchmark outputs immutable once frozen.
